## Context

Today the usage story is scattered (repo README, debugpy module README, launcher
comments, VS Code configs). The end-user deliverable is ONE guide with one mental
model — resolve target → ensure runtime → sync → launch → attach — across the
three transports. The honesty constraint is the point: the old wrapper's docs and
variant names promised capabilities the firmware didn't have; this guide must
never claim what the runtime probe (`caps` in the MPDBG-READY handshake) would
contradict.

## Scope

In scope: one user-facing guide in this repo (and/or the debugpy module README
where content upstreams with s8.2), covering unix / serial / network transports,
the `mpdebug.toml` target model, firmware fetch vs build, and capabilities.
Out of scope: contributor/integration docs (CLAUDE.md, mbm) — already current;
extension marketplace copy (EPIC-7).

## Design constraints

- No capability claim the probe contradicts (STORY-8.4 acceptance; D4).
- Serial DAP is documented as multi-CDC-only, network as the primary device
  transport (D3/Q3); single-UART boards: network required, framing spike deferred.
- v1 board set for prebuilt firmware: unix, rpi_pico_w, PYBD_SF6, ESP32_GENERIC (Q2).
- User CLAUDE.md language rules: factual, conservative; no "comprehensive".
- tty devices referenced as `/dev/serial/by-id/<name>` in every example.

## Approach sketch

Structure: (1) 5-minute quick start on unix (zero hardware); (2) the mental model
(one diagram: resolve → ensure → sync → launch → attach; control plane vs data
plane); (3) per-transport pages sharing the same headings; (4) firmware: fetch
(manifest-verified) vs build (docker per s3.2); (5) capabilities: what the probe
reports and what each value means in the editor (greyed-out locals, placeholder
names); (6) troubleshooting keyed on actual failure modes (stale device code →
staleness guard message; no `stopped` event → firmware without settrace, probe
output shown). Write against the real commands; every snippet must be paste-able
and tested.

## Acceptance criteria and tests
