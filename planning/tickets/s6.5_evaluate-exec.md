## Context

Surfaced by SPIKE-2.1's follow-ups: since locals are read-only (D4 — no
`frame._set_local` in any firmware) and mount enables live-source iteration, the
remaining live-edit escape hatch is executing *statements* from the Debug Console —
redefine a function, mutate a global, monkey-patch a module — which takes effect on
the next call. Today `evaluate` only `eval()`s expressions, so `x = 5` or
`def f(): ...` fails with a syntax error.

## Scope

In scope: statement execution for DAP `evaluate` requests whose `context` is
`repl` or `clipboard`; expression behaviour for `watch`/`hover` unchanged.
Out of scope: local-variable write-back (impossible without firmware support — D4),
multi-frame exec (top frame's globals only, matching the existing setVariable
restriction), autocomplete/completions requests.

- `micropython-lib/python-ecosys/debugpy/debugpy/server/pdb_adapter.py:649-669` —
  `evaluate_expression(self, expression, frame_id=None)`: resolves the frame from
  `self.variables_cache[frame_id]` (else `self.current_frame`), builds
  `globals_dict`/`locals_dict` from `f_globals`/`f_locals` (hasattr-guarded), then
  `eval(expression, globals_dict, locals_dict)` at :666.
- `micropython-lib/python-ecosys/debugpy/debugpy/server/debug_session.py:458-474` —
  `_handle_evaluate`: extracts `expression` and `frameId`; the DAP `context` field is
  currently read NOWHERE (a commented-out `# context = args.get("context", "watch")`
  at :462). This handler is where context-based dispatch goes.
- Response is `{"result": str(result), "variablesReference": 0}` (:470-471) — for a
  statement there is no value; return `""` or a short confirmation, and keep
  `variablesReference: 0`.
- Constraint carried from BACKGROUND: `f_locals` is a snapshot dict — passing it to
  `exec` means local assignments are silently ineffective. Do NOT pretend otherwise:
  statements execute against `f_globals` (live) with the locals snapshot readable.

## Design constraints

- No threads, no blocking: exec happens synchronously inside the trace-callback
  poll loop like every other request — a long-running statement blocks the session;
  that is accepted and documented (same as a long `evaluate` today).
- Honesty rule: a statement that assigns to a name shadowed by a paused frame's
  LOCAL must not silently appear to work. Detect the case (name in locals snapshot)
  and include a warning in the result string.
- Security posture is unchanged: `evaluate` is already remote `eval()` by design
  (BACKGROUND "rough edges" #6); exec adds no new class of exposure.
- Upstream-friendly: lands in the micropython-lib debugpy lineage (#1022 follow-up).

## Approach sketch

In `_handle_evaluate`, read `context = args.get("context", "watch")`. For
`repl`/`clipboard`: try `eval` first (so `1+1` still prints a value in the console);
on `SyntaxError`, `exec(expression, globals_dict)` — MicroPython supports
`compile(..., "exec")`/`exec` on the unix and device builds in use (settrace builds
force `MICROPY_PY_BUILTINS_CODE_FULL`). For `watch`/`hover`: exactly today's path.
Add the shadowed-local warning check before returning.

- [x] From the Debug Console while paused: `x = 42` (global) mutates the running
  program (verified after continue); `def f(): return 1` rebinds `f` for the next
  call.
- [x] `watch`/`hover` contexts still reject statements exactly as before
  (parametrised regression tests, plus context-absent defaults to watch).
- [x] Assigning a name that exists in the paused frame's locals yields a visible

- [x] A statement raising an exception returns `success=False` with the message,
  session continues usable.

  requests (`tests/dap.py` gains a context-aware `evaluate` helper;
  `tests/test_dbg_06_evaluate.py`, 7 tests). Suite: 46 passed, 1 xfailed.

Notes carried forward:
- Statements execute against `f_globals` only, so a statement that READS a
  paused frame's local raises a NameError that can mislead (the honesty rule
  covers writes, not reads). Candidate follow-up for the Debug Console UX
  docs (s8.4) or a merged-namespace read approach if it ever matters.
- The shadowed-local warning contract (documented in the code) covers bare and
  augmented assignment only; unpacking / attribute / def-class rebinds pass
  unwarned. On save_names=false firmware the warning cannot fire
  (placeholder keys) — documented limitation.
- s8.2 note: the delivered docstrings drew a snapshot-coherence review nit
  (some contract text reads like change narration) — tidy when the
  foundations branch is squashed for upstream.
- The host harness has a pre-existing startup-handshake timing race that can
  intermittently fail unrelated tests under system load (seen ~1 in 5 full
  runs here and in s3.3's runs; a `stopped` event fires while the debuggee is
  still in `wait_for_client`). Worth a dedicated fix pass with s5.5.
