# extension: Status-bar target picker over `mpdebug.toml` with honest capability display

Phase: EPIC-7 / STORY-7.2
Depends on: s7.1 (the provider the picker feeds), s5.2 (`mpdebug.toml` named-target format — that ticket owns the schema)
Written: 2026-07-06 at HEAD 59fab8450a (micropython 18c76f7c56, micropython-lib 0d42fcd5bc)

## Context

With s7.1 in place, F5 debugs "the target" — this ticket supplies the UI for choosing
which target, and surfaces what the chosen firmware can actually do. Honest capability
display is a settled design rule (D4, STORY-1.2): capability comes from the runtime
probe, never from a variant/target name. `ROADMAP.md` STORY-7.2. STORY-7.3 (collapse
legacy launch.json/tasks.json) is inline-only in the roadmap (trivial) and lands with
s7.1, not here.

## Scope

In scope: a status-bar item listing the named targets parsed from the project's
`mpdebug.toml`; the selection persists (workspaceState) and drives which target s7.1's
resolver launches; after a session's handshake arrives, the probed capability summary
is shown honestly (e.g. "locals: placeholders", "local edit: unsupported").

Out of scope: the `mpdebug.toml` schema and resolution semantics (owned by s5.2 in the
mpremote command — the extension only *reads* target names, it never resolves
transports); editing/creating targets from the UI (defer; users edit the TOML).

## Files and anchors

- `mpdebug.toml` — format owned by ticket s5.2 (`kind = unix|serial|network`, connect
  string via `/dev/serial/by-id/...`, firmware/capability requirement). The picker
  needs only the target names + `kind` for display; re-resolve this anchor at phase
  entry once s5.2 has landed.
- `caps` dict fields, verified against
  `micropython-lib/python-ecosys/debugpy/debugpy/server/debug_session.py:82`
  (`probe_capabilities()`): `settrace`, `save_names`, `set_local`, `f_back` — booleans.
  Reaches the extension in the `MPDBG-READY` payload
  (`launcher/mpy_launch_debugpy.py:90`) parsed by s7.1's handshake parser.
- Display mapping (capability → user-facing text):
  `save_names: false` → "locals: placeholder names (local_N)";
  `set_local: false` → "local edit: unsupported (globals editable)";
  `settrace: false` → target not debuggable — picker marks it and F5 fails early
  with the firmware guidance message.
- New code: `extension/src/targetPicker.ts` + wiring in `extension/src/extension.ts`
  (s7.1's tree).

## Design constraints

- Never display a capability the probe would contradict (STORY-3.4/8.4 AC language);
  before the first handshake of a session the picker shows the target as "capabilities
  unprobed", not optimistic defaults.
- The extension does not parse/validate transport details from `mpdebug.toml` beyond
  names and `kind` — resolution is the command's job (D1). A missing/ambiguous file
  shows a actionable message ("no mpdebug.toml — run `mpremote debug --init`" or the
  s5.2-equivalent), not a stack trace.
- Selection must also be settable per-launch via a `target` attribute in a
  launch.json config (picker is the default, not the only path).

## Approach sketch

`vscode.window.createStatusBarItem` showing `$(debug-alt) <targetName>`; click opens a
QuickPick built from the parsed TOML (`kind` as description). Store the pick in
`workspaceState`. s7.1's resolver reads the stored pick when the launch config has no
explicit `target`. After s7.1 parses a handshake, publish the caps via an event; the
picker appends a one-line capability summary to the status-bar tooltip and (first time
per target per session) a subtle notification if `save_names` is false — the "you are
seeing placeholder names" surprise is the one worth interrupting for. Parse TOML with
a small vendored parser or `@iarna/toml` (record the choice in the PR).

## Acceptance criteria and tests

- [ ] picker lists exactly the targets in `mpdebug.toml`; selection drives F5.
- [ ] selected target survives a window reload (workspaceState).
- [ ] capability summary appears after attach and matches the handshake caps verbatim
  (test with a `save_names: false` firmware — placeholder wording shown).
- [ ] no `mpdebug.toml` / malformed TOML → actionable message, F5 still usable with an
  explicit `target` attribute.
- [ ] adversarial: a target whose name lies (e.g. "set_local" in the name) displays
  only probed truth.
