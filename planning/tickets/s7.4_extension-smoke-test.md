# extension/CI: Extension-host smoke test — resolver yields a valid attach and reaches a breakpoint on unix

Phase: EPIC-7 / STORY-7.4
Depends on: s7.1 (the provider under test); built unix firmware + working `mpremote debug` (EPIC-5) available in CI
Written: 2026-07-06 at HEAD 59fab8450a (micropython 18c76f7c56, micropython-lib 0d42fcd5bc)

## Context

EPIC-7's provider (s7.1) is glue between three moving parts (mpremote child process,
handshake contract, ms-python.debugpy attach); glue is where regressions hide. The
repo's testing rule (`CLAUDE.md`): tests drive real behaviour, not source-string
checks, and stay machine-independent. STORY-7.3 is inline-only in the roadmap
(trivial, lands with s7.1); it needs no test beyond "the old hacks are gone".

## Scope

In scope: an automated extension-host test (`@vscode/test-electron`) that (a) unit-tests
the handshake parser as pure functions, and (b) end-to-end: starts a `micropython`-type
debug config against a unix target, asserts the resolver spawned the command, parsed
the handshake, started a `debugpy` attach session, and a breakpoint in
`src/target.py` is hit. Headless CI wiring (xvfb on linux).

Out of scope: device-transport tests (s6.4 owns hardware-in-loop), picker UI tests
beyond selection persistence (s7.2's unit tests), performance.

## Files and anchors

- `extension/src/handshake.ts` (created by s7.1) — the pure parser under unit test;
  contract: consume exactly one `MPDBG-READY {json}` line
  (`launcher/mpy_launch_debugpy.py:90` — fields `host`, `port`, `caps` with boolean
  `settrace`/`save_names`/`set_local`/`f_back`), ignore all other stdout noise
  (banner text precedes it, `:70-78`).
- `src/target.py` — the sample debuggee the E2E test breaks in; the existing host
  harness (`tests/test_epic1_foundations.py:109`) breaks at line 80 because the
  `line` event fires BEFORE the statement executes — reuse the same line-choice
  rule (break after an assignment, never on it).
- `tests/` — the existing pytest DAP harness (fake VS Code) is the behavioural
  reference for what a correct attach/stop looks like; the extension test asserts the
  same observable outcome through the real VS Code debug API
  (`vscode.debug.onDidStartDebugSession`, `onDidChangeActiveDebugSession`,
  breakpoint hit via `vscode.debug.activeDebugSession` + DAP `stopped` tracker).
- New code: `extension/src/test/` (test runner + suites), CI job in the repo's
  workflow (xvfb-run; cache the VS Code download).

## Design constraints

- Machine-independent: no absolute paths; firmware located the same way the pytest
  harness does (`MPY_DEBUG_FIRMWARE` env var, falling back to
  `micropython/ports/unix/build-standard/micropython` — see
  `tests/fixtures/debuggee.py`).
- The E2E test must fail (not skip) if `ms-python.debugpy` is missing — that IS the
  dependency declaration under test. Install it into the test host
  (`--install-extension ms-python.debugpy`) in the runner.
- No sleeps for synchronisation: await debug-API events with bounded timeouts
  (mirror the server's 30 s `WAIT_FOR_CLIENT_TIMEOUT_S`).

## Approach sketch

Two layers. Unit: mocha suites over the handshake parser (valid line, noise-wrapped
line, split-across-chunks line, garbage JSON, no line → timeout error). E2E: a
`DebugAdapterTrackerFactory` registered for `debugpy` records DAP events; the test
sets a breakpoint in `src/target.py`, calls
`vscode.debug.startDebugging(ws, {type: "micropython", name: "test", request: "launch", target: "unix", ...})`,
then awaits the tracker seeing a `stopped` event with reason `breakpoint`, then
disconnects and asserts the mpremote child exited. CI: linux job, xvfb-run,
`make firmware-unix` prerequisite shared with the existing test job.

## Acceptance criteria and tests

- [ ] parser unit suite green, including the adversarial inputs above.
- [ ] E2E: resolver-started session reaches a `stopped`/`breakpoint` on unix with zero
  hand config; child process cleaned up after disconnect.
- [ ] suite green headless in CI (xvfb), using the same firmware artifact as
  `make test`.
- [ ] a deliberately broken handshake (env-var fault injection or a stub command)
  produces the s7.1 diagnostic error path, asserted.

## Workflow shape

Test authoring on haiku, with sonnet for the extension-host runner/CI plumbing if it
grows; opus review focuses on flakiness (event races, timeouts) — flaky debug-API
tests are worse than none.
