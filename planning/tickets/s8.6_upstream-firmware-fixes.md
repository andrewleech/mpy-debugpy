## Files and anchors

- Fix (a) `33721cfe4` "py/profile: Fix local variable slot indexing in
  frame_f_locals." — `py/profile.c` (+40/−20). Without it, locals are reported
  under WRONG names (slot-assignment heuristic bug).
- Fix (b) `e5b5fa9a7` "unix/standard: Disable MICROPY_PY_SYS_SETTRACE_LOCALNAMES_PERSIST
  by default." — `ports/unix/variants/standard/mpconfigvariant.h` (+6/−1). The
  in-tree comment states the root cause: the code-info writer appends data
  unconditionally without teaching the generic line-number reader to skip it,
  corrupting `sys.settrace()` line numbers for ALL compiled code (see
  `py/emitbc.c mp_emit_bc_start_pass()` and `py/profile.c mp_prof_bytecode_lineno()`).
- Follow-up fixup `18c76f7c5` (missing trailing newline in
  `tests/misc/sys_settrace_localnames.py.exp`) — squash into the tests commit when
  rebuilding the branch, or carry as-is.
- Canonical target branch: `andrewleech/local_names_implementation`, whose locals
  commits (`6a0b51bc2`, `d96a0780c`, `971f9ad27`, `2ff9f3cd8`) are 1:1 with the
  integration's cherry-picks — but the branch sits on stale ancestry
  (`872b5d9c5`/`8d3dce03b`/`a05fa0582` old gettrace/_getframe work that #8767
  `3280c2a5e` now supersedes). Rebuild it (or a successor branch) as: the 4 locals
  commits rebased onto `andrewleech/pdb_support` (#8767) + fix (a) + fix (b) + the
  new regression test. Coordinate the branch-name choice with s8.5.

## Approach sketch

Rebuild the canonical locals branch on the fork (see anchors), force-push, open/
refresh the fork PR per the default-to-upstream policy. Add the regression test as
its own commit on that branch. The full test sketch is in
`../20260706_flocals_params.md` §3 — key points: `tests/misc/` convention (SKIP
guard on missing `sys.settrace`, committed `.exp`); on this branch `line` fires
BEFORE the statement executes, so the tracer must key on `'a' in frame.f_locals`
rather than a fixed line, and the load-bearing assertion is the param subset
(`a`, `b`, `c` present under real names) — the body-local is secondary and may be
dropped if CI timing proves brittle.
