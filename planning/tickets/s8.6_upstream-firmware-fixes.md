## Approach sketch

Rebuild the canonical locals branch on the fork (see anchors), force-push, open/
refresh the fork PR per the default-to-upstream policy. Add the regression test as
its own commit on that branch. The full test sketch is in
`../20260706_flocals_params.md` §3 — key points: `tests/misc/` convention (SKIP
guard on missing `sys.settrace`, committed `.exp`); on this branch `line` fires

rather than a fixed line, and the load-bearing assertion is the param subset
(`a`, `b`, `c` present under real names) — the body-local is secondary and may be
dropped if CI timing proves brittle.

## Execution outcome (2026-07-15)

Delivered as `andrewleech/local_names_implementation` @ `7ae5f769c` (was
`2ff9f3cd8`; local backup branch `local_names_implementation-pre-rebuild`),
7 commits on `pdb_support` (`3280c2a5e`); fork PR #5 (base `pdb_support`)
carries it. Executed by two multi-agent workflows (implement→test→review
looped); the first pass's adversarial review refuted the faithful-replay claim,
which led to the deliberate divergences below — all reviewed, all verified:

- The original locals commits shipped defects the recomposition never surfaced
  because these tests were never run: 3 of the misc locals tests fail as
  composed (stale `local_NN` expectations + CPython 3.13+ `FrameLocalsProxy`
  divergence), `tests/basics/sys_settrace_localnames_comprehensive.py` fails
  its dynamic CPython comparison (MicroPython legitimately emits one extra
  `line` event on a for-loop's first iteration — bytecode line-table artifact,
  reproduced on the unmodified base), and a 276-line scratch harness
  (`test_local_names.py`) plus a planning doc were committed in-tree.
- Real sentinel bug fixed: `MP_QSTR_NULL` (the interned qstr for the string
  "NULL", nonzero) was used where `MP_QSTRnull` (index 0, the value
  `m_new0`-initialised `local_names` slots actually hold) was meant, in
  py/emitglue.h, py/profile.c and py/persistentcode.c — unnamed slots leaked
  under an invalid key instead of falling back to `local_NN`. The integration
  branch still carries this bug until s8.5 recomposes.
- `frame_f_locals` semantics decision (orchestrator-settled): keep the
  integration's runtime fallback (no name data ⇒ physical-slot `local_NN`
  dump) — the first workflow's compile-time rework would have returned `{}`
  for `.mpy`-loaded code under LOCALNAMES builds, regressing the device
  debugging story. Final structure = integration's, plus sentinel fix, plus
  `#if` guards widened to `LOCALNAMES || PERSIST` (fixes two pre-existing
  compile failures: PERSIST-only builds, and SETTRACE-only builds after the
  restructure). Four-way feature-macro build matrix now compiles.
- Per-commit sweep: every one of the 7 commits builds and passes every
  settrace test that exists at that commit (bisect-clean). verifygitlog ok
  across the range; cherry-pick provenance lines stripped; Jos's authorship
  and sign-offs preserved; missing sign-offs added
  (andrew.leech@planetinnovation.com.au, matching pdb_support).
